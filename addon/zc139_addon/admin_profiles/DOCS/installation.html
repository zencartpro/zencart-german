Installation und Konfiguration


Voraussetzungen

Dieses Modul ist nur für Zen-Cart 1.3.9 deutsch geeignet

Für den Einbau dieses Moduls sollten Sie über folgende Werkzeuge verfügen:

1) Ein guter Texteditor.
Damit ist nicht das in Windows enthaltene Notepad oder Microsoft Word gemeint.
Sie benötigen einen Texteditor, der utf-8 versteht und auch im Format utf-8 ohne BOM abspeichern kann.
Meine Empfehlung: UltraEdit (kostenlose 30 Tage Testversion verfügbar)
Ebenfalls gut geeignet ist der kostenlose Texteditor Notepad++

2) Ein Tool zum Vergleichen von Dateien
Bei der Installation dieses Moduls ist es erforderlich, dass Sie den Inhalt einiger Ihrer bestehenden Zen-Cart Dateien mit dem Inhalt der neuen Moduldateien vergleichen und die Änderungen zusammenführen ("mergen").
Meine Empfehlung: BeyondCompare (kostenlose 30 Tage Testversion verfügbar)
Ebenfalls gut geeignet ist das kostenlose Programm WinMerge


Empfehlung:

Dieses Modul erst in einem Testsystem einbauen und dort konfigurieren/testen und endgültig an die eigenen Wünsche anpassen. Erst dann in einem Liveshop einsetzen! Sichern Sie unbedingt alle Dateien Ihres Shops per FTP und sichern Sie die Datenbank mit phpMyAdmin oder anderen geeigneten Tools!

================= WICHTIG =====================
Vor dem Einbau dieser Änderungen:
BACKUP von Shop und Datenbank machen!
Keine Haftung, Verwendung auf eigene Gefahr!
BACKUP gemacht? Ok, dann weiterlesen...
===============================================


Wichtiger Hinweis:
Beim Einbau dieses Moduls ist es zwingend notwendig, dass Sie die Reihenfolge der im folgenden beschriebenen Installationsschritte einhalten.
Als erstes wird die Datenbankänderung eingespielt!


1.
Im Order SQL die Datei install.sql mit einem Texteditor öffnen und den Inhalt kopieren.
Wichtig:
Normalerweise hat der Admin, der alle Rechte hat (Superuser) in Zen-Cart die ID 1
Überprüfen Sie unter Tools > Administratoren, ob Ihr User wirklich die ID 1 hat.
Sollte Ihr User eine andere ID haben, dann ändern Sie folgende Zeile in der install.sql entsprechend ab:

SET @superuser = 1;

2.
Zen-Cart Administration öffnen und unter Tools > SQL Patches installieren den Inhalt ins Feld hineinkopieren und absenden.
Dadurch werden die nötigen neuen Tabellen in der Datenbank angelegt.

3.
Im Ordner NEUE DATEIEN dieses Downloads den Ordner DEINADMIN auf den Namen Ihres Adminverzeichnisses umbenennen.
Dann in der vorgegebenen Struktur hochladen, dabei werden keine bestehenden Dateien überschrieben.

4.
Im Ordner GEAENDERTE DATEIEN dieses Downloads den Ordner DEINADMIN auf den Namen Ihres Adminverzeichnisses umbenennen.
Achtung:
Beim Hochladen dieser Dateien werden bestehende Dateien überschrieben!
Daher:
Falls bereits einmal Änderungen an diesen Dateien in der Zen-Cart Installation gemacht wurden, mit WinMerge oder BeyondCompare die Dateien vergleichen und nur die Änderungen in die bestehenden Dateien übernehmen.
Wurden noch keine Änderungen gemacht (Zen-Cart wurde z.B. gerade erst frisch installiert), die Ordner/Dateien aus dem Ordner GEAENDERTE DATEIEN einfach in der vorgegebenen Struktur in Ihr Zen-Cart Verzeichnis hochladen.

5.
Loggen Sie sich nun als Admin ein. (Sollte es das erste Mal sein, dass Sie sich als Admin einloggen müssen Sie sich mit dem Adminuser einloggen, der die ID 1 hat).
Gehe Sie auf Tools > Administratoren
Klicken Sie auf das Rechte Symbol (weißes "P" in einem orangenen Kreis) für den Admin, für den Sie die Rechte verwalten möchten.
Am besten machen Sie das zuerst bei Ihrem eigenen Adminuser (der mit der ID 1), der alle Rechte haben soll.

Hier finden Sie nun zwei Ebenen der Verwaltung:
Blau unterlegt die Hauptmenüpunkte (Configuration Menu, Catalog Menu, Modules Menu, Customers Menu, usw.)
Es werden hier immer die englischen Bezeichnungen angezeigt, da Admin Profiles leider nicht multilanguagefähig ist.
Diese Punkte entsprechen einfach dem deutschsprachigen Hauptmenü, das Sie aus Ihrem deutschen Zen-Cart Shop gewohnt sind (Konfiguration, Webshop, Module, Kunden, usw.)

Sie können also die Hauptmenüpunkte sichtbar oder unsichtbar machen, indem Sie die Checkbox neben dem Namen des Hauptmenüs auswählen oder deaktivieren und danach auf Speichern gehen. Versuchen Sie es an Ihrem eigenen Adminuser und Sie werden die Änderung sofort erkennen.
Nichtsdestotrotz kontrolliert dies weder die Verfügbarkeit der Untermenüpunkte noch, dass die Benutzer diese sehen können. Diese können weiter durch die Eingabe der exakten URL von Admins erreicht werden oder auch durch jede spezifische Seite die Sie hierfür definieren.

Die Untermenüpunkte müssen Sie also extra ein- oder ausschalten durch das Ein- oder Ausklicken der Checkbox neben jeder Seitenbeschreibung. Auch hier werden wieder - da nicht anders möglich - die englischen Beschreibungen angezeigt.
Probieren Sie auch das nun bei Ihrem eigenen Admin aus und deaktivieren Sie z.B. die Checkbox bei E-Mail Options und clicken auf Speichern. Rufen Sie dann Konfiguration > E-Mail Optionen auf und Sie werden sehen, dass Sie auf diese Einstellungen keinen Zugriff mehr haben. Schalten Sie sich also gleich danach besser wieder dafür frei :-)

Falls Sie bereits Zusatzmodule installiert haben, sollten diese ganz unten unter Third Party Mods aufscheinen.
Kreuzen Sie unbedingt alle an, damit Sie auch auf diese Konfigurationseinstellungen immer Zugriff haben!

WICHTIG:
Entfernen Sie bei Ihrem eigenen Admin NIE den Hauptmenüpunkt Tools oder den Untermenüpunkt admin! Sie haben sonst keine Möglichkeit mehr, sich die Rechte wieder zu geben!
Generell, sollten Sie bei Ihrem eigenen Admin darauf achten, dass immer ALLES angekreuzt und somit erlaubt ist

Nachdem Sie sich mit Ihrem eigenen Adminuser mit der Funktionalität vertraut gemacht haben, stellen Sie nun die Berechtigungen für Ihre weiteren Administratoren so ein, wie Sie es wünschen.

6.
Wichtiger Hinweis für zusätzliche Erweiterungen, die Sie später installieren
Sobald Sie Admin Profiles im Einsatz haben, müssen Sie darauf achten, dass Sie nach jeder späteren Installation einer Erweiterung, diese für sich selbst freigeben.
Bsp:
Sie installieren das Lightbox-Modul. Das legt unter Konfiguration einen Untermenüpunkt Lightbox an, in dem die Einstellungen der Lightbox zu finden sind.
In den Admin Profiles Einstellungen wird die Lightbox dann ganz unten unter Third Party Mods aufscheinen und nicht angekreuzt sein.
Dies gilt für alle Erweiterungen, die später installiert werden: Nicht vergessen, sie immer für den eigenen Superadmin freizugeben!

Admin Profiles vollständig deinstallieren

Im Ordner SQL dieses Downloads befindet sich eine uninstall.sql
Zen-Cart Administration öffnen und unter Tools > SQL Patches installieren den Inhalt ins Feld hineinkopieren und absenden. Dadurch werden die 4 von Admin Profiles angelegten Tabellen gelöscht.
Danach alle neuen von diesem Modul angelegten Dateien vom Server löschen.
Alle von diesem Modul geänderten Dateien wieder mit den Originaldateien bzw. den Dateien Ihres vorher angelegten Backups ersetzen
