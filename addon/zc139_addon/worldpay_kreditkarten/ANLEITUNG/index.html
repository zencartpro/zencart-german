Sinn und Zweck dieses Moduls

Mit diesem Modul kann Kreditkartenzahlung via WorldPay angeboten werden.
Die Einstellungsmöglichkeiten sind im deutschen Admin auf deutsch und im englischen Admin auf englisch.
Alle Einstellungsmöglichkeiten sind dort erklärt.
Nach der Installation ist in der Zen-Cart Administration unter Kunden der Menüpunkt WorldPay Zahlungen vorhanden, in dem alle per WorldPay getätigten Zahlungen sowie Rückmeldungen von WorldPay ersichtlich sind.

Dieses Modul ist nur für eine deutsche Zen-Cart 1.3.9 Version geeignet

Voraussetzung ist eine Anmeldung zu Business Gateway Plus bei WorldPay.
Detaillierte Informationen dazu auf: http://www.worldpay.com/deutsch/ecom/index.php?page=service&sub=worlddirect
Sobald ein Vertrag mit WorldPay abgeschlossen wurde, steht bei WorldPay ein Testsystem für Test-Transaktionen zur Verfügung, das auch von diesem Modul unterstützt wird. Es können also erst alle Einstellungen im Testsystem vorgenommen, im Modul darauf angepasst und Testbestellungen durchgeführt werden. Später kann dann auf Live-Transaktion umgestellt werden.

Installation

Voraussetzungen


Dieses Modul ist nur für die deutsche Zen-Cart 1.3.9 Version geeignet

Für den Einbau dieses Moduls sollten Sie über folgendes Werkzeug verfügen:

1) Ein guter Texteditor
Damit ist nicht das in Windows enthaltene Notepad oder Microsoft Word gemeint.
Sie benötigen einen Texteditor, der utf-8 versteht und auch im Format utf-8 ohne BOM abspeichern kann.
Meine Empfehlung: UltraEdit (kostenlose 30 Tage Testversion verfügbar)
Ebenfalls gut geeignet ist der kostenlose Texteditor Notepad++

Empfehlung:

Dieses Modul erst in einem Testsystem einbauen und dort konfigurieren/testen und endgültig an die eigenen Wünsche anpassen. Erst dann in einem Liveshop einsetzen! Sichern Sie unbedingt alle Dateien Ihres Shops per FTP und sichern Sie die Datenbank mit phpMyAdmin oder anderen geeigneten Tools!

WICHTIG
Vor dem Einbau dieser Änderungen:
BACKUP von Shop und Datenbank machen!
Keine Haftung, Verwendung auf eigene Gefahr!
BACKUP gemacht? Ok, dann weiterlesen...

1.
Im Order NEUE DATEIEN den Ordner DEINADMIN auf den Namen Ihres admin Verzeichnisses umbenennen.

2.
Dann die Ordner/Dateien in der vorgegebenen Struktur in Ihr Zen-Cart Verzeichnis hochladen. Dabei werden keine bestehenden Dateien überschrieben.

3.
In der Zen-Cart Administration unter Module > Zahlungsarten das neu erschienene WorldPay Modul markieren. Auf "Installieren" und danach die nötigen Einstellungen vornehmen.

4.
Tip: Das Modul ist nach der Installation auf Testumgebung geschaltet. Richten Sie erst Ihre WorldPay Testumgebung ein und vergeben dann in der Zen-Cart Administration die zur Testumgebung passenden Einstellungen. Ein großer Vorteil von WorldPay ist die Testumgebung, nutzen Sie sie solange, bis sie wirklich mit allen Einstellungen zufrieden sind!
Sobald WorldPay den Testaccount freigegeben hat, erhält man die nötigen Logindaten für die WorldPay Administration.
Hier die gewünschten Einstellungen vornehmen und das Zen-Cart Modul entsprechend einstellen.
Wichtig ist in der WorldPay Administration, dass die Callback URL exakt so eingegeben wird:
https://<wpdisplay item="MC_callback">
Sollte man im Shop kein SSL verwenden (nicht empfohlen!), wäre es:
http://<wpdisplay item="MC_callback">
Eine ausführliche Dokumentation zu den Einstellungen in der WorldPay Administration bietet:
http://www.worldpay.com/deutsch/support/kb/bg/testandgolive/tgl.html

6.
Testbestellungen im Testshop machen (Dummy-Kreditkartennummern für die Testumgebung) liefert
http://www.worldpay.com/deutsch/support/kb/bg/testandgolive/tgl.html

7.
Zahlungsseite falls gewünscht in der WorldPay Administration anpassen

8.
Bevor die Liveumgebung genehmigt wird, muss WorldPay selbst eine Testbestellung machen um zu prüfen, ob alles richtig integriert wurde und die erforderlichen Layoutanpassungen (z.B. WorldPay Logo) gemacht wurden.
Eine Übersicht der Website-Richtlinien bietet:
http://www.worldpay.com/deutsch/support/index.php?page=start&sub=website&subsub=rules
Diese Testbestellung kann man in der WorldPay Administration beantragen. Hier dann die Adresse des Testshops angeben.
Sobald WorldPay die Testbestellung gemacht hat und alles passt, geben sie die Liveumgebung frei.

9.
Modul im Liveshop installieren, entsprechend konfigurieren, Layoutänderungen aus dem Testshop übernehmen, auf Livebetrieb schalten und los gehts.
