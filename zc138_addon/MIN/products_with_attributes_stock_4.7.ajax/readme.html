Stock by Attributes v4.7ajax

Übersetzung von MaleBorg

Diese Erweiterung basiert auf der Zen-Cart Contrib von dafonz (products_with_attributes_stock), welche von danielcor für Zen Cart Version 1.3 angepasst wurde. Weitere Anpassungen für Zen-Cart 1.3.5 und höher wurden von kuroi vorgenommen. Hugo13 hat dann der aktuellen Version Ajax Funktionalität hinzugefügt.

Anmerkung: In der 4.7 wurde lediglich eine kleine Modifikation vorgenommen. Diese erlaubt das gleichzeitige Anlegen von Lagerbeständen für mehrere Attributkombinationen.

Anmerkung2: In der vorliegenden Version 4.7ajax wurden Ajax Funktionen hinzugefügt.

Diese Version 4.7ajax ist speziell für die Zen-Cart Version 1.3.8 angepasst und sollte daher nicht für andere Versioenn benutzt werden.

Für die Zen-Cart Version 1.2.5 - 1.2.7d nutzen Sie bitte die Contrib "products_with_attributes_stock" von dafonz.
Für die Zen-Cart Version 1.3.0 - 1.3.0.2 nutzen Sie bitte die Contrib "Stock Attributes by danielcor".
Für die Zen-Cart Versionen 1.3.5 nutzen Sie bitte die Version 4.2 von dieser Erweiterung,
für Zen-Cart Version 1.3.6 nutzen Sie bitte die Version 4.3 von dieser Erweiterung.
Die Versionen 4.4, 4.5 und 4.6 dieser Erweiterung sind mit der Zen-Cart Version 1.3.7 und 1.3.7.1 kompatibel.

Was macht diese Erweiterung?

Diese Erweiterung ermöglicht Ihnen einen Lagerbestand für jede Artikel-Attribut Kombination zu hinterlegen.

Zum Beispiel: Artikel X bieten Sie in den Größen S, M und L an. Bisher konnten Sie in der Lagerverwaltung nur entnehmen, welche Menge von Artikel X im Lager ist, aber nicht welche Menge von welcher Größe vorhanden ist.
Dank dieser Erweiterung ist das nun möglich. Sie können ebenfalls für Kombinationen aus mehreren Attributen Lagerbestände hinterlegen.
Zum Beispiel: Artikel Y bieten Sie in den Größen S, M und L und in den Farben Grün, Rot und Blau an. Sie können nun für alle möglichen 18 Kombinationen von diesen Attributen Lagerbestände hinterlegen.

Unterschiede zu vorherigen Versionen dieser Erweiterung

Vorherige Versionen dieser Erweiterung zwangen Sie nach der Installation dazu, das der Lagerbestand des Artikels mindestens so groß war wie der Lagerbestand des Attributs. Ansonsten wurde Artikel auf "OUT OF STOCK" gestellt. Ab Version 4.6 dieser Erweiterung haben Sie jetzt für jeden Artikel die Möglichkeit die Lagerverwaltung auf den Artikellagerbestand oder den Lagerbestand der Artikel-Attribut Kombination reagieren zu lassen.

Wenn Sie einen Lagerbestand für eine Artikel-Attribut Kombination eingeben, werden alle Lagerbestandsberechnungen aufgrund dieses Lagerbestands durchgeführt. Attribute für den keine Lagerbestand hinterlegt wurde, werden so behandelt, als hätten Sie einen Lagerbestand von 0. Wenn Sie keinen Lagerbestand für die Attribute angelegt haben, oder Sie löschen die angelegten Lagerbestände für die Attribute, dann wird bei der Lagerbestandverwaltung wieder auf den Gesamtlagerbestand des Artikels zurückgegriffen.

Es wurden Ajax Funktionen hinzugefügt.

  • Der Suchfilter "%ma" filtert alle Artikel mit "ma" im Artikelnamen oder der Artikelnummer; das Suchergebnis wird in einem Cookie gespeichert.
  • Ein Klick auf den "Lagerbestandswert" erstellt ein Eingabefeld in das Sie den neuen Wert eingeben können; der STORE Button speichert alle Änderungen ab.

Installation

Wie bei jeder Installation einer Erweiterung sollten Sie unbedingt VORHER ein Backup machen.

Spielen Sie als Erstes die beiliegende SQL Datei 'stock_attribute_german.sql' ein. Dazu gehen Sie in den Adminbereich --> Tools --> SQL Patches installieren. Dort können Sie die Datei entweder uploaden oder Sie kopieren den Inhalt der SQL Datei in das Textfeld und klicken auf senden. Sollten Sie eine andere Methode für das Ausführen von SQL Dateien bevorzugen, denken Sie bitte daran, das ein evtl. benutztes Präfix in der SQL Datei einzutragen.

Bereiten Sie den Upload der Ordner und Dateien für Ihren Server vor. Dazu müssen Sie in folgenden Ordnern (includes/modules/YOUR_TEMPLATE und includes/templates/YOUR_TEMPLATE) YOUR_TEMPLATE durch den Namen von ihrem zur Zeit benutzten Template ersetzen.

Sollten Sie bisher keine Veränderungen an Ihrer Zen-Cart Version vorgenommen haben, dann können Sie die Ordner (admin, includes, ajax) per FTP Programm (z.B. FileZilla) in Ihr Zen-Cart Hauptverzeichnis hochladen.

Es werden dabei sechs Zen-Cart Dateien überschrieben. Sollten Sie bereits andere Erweiterungen installiert haben oder Ihr Zen-Cart bereits selber verändert haben, sollten sie überprüfen, ob bereits die folgenden Dateien verändert wurden:

  • admin/includes/stylesheet.css
  • admin/orders.php
  • includes/classes/order.php
  • includes/functions/functions_lookups.php
  • includes/modules/pages/checkout_shipping/header_php.php
  • includes/modules/pages/shopping_cart/header_php.php

Falls Sie oder irgendwelche Erweiterungen diese Dateien bereits verändert haben, dann müssen Sie die notwendigen Änderungen in diesen Dateien selber durchführen. Hilfreich sind dabei Dateivergleichsprogramme wie WinMerge.

Upgrade von vorherigen Versionen

Um von Version 4.x auf Version 4.7ajax upzugraden ist es am einfachsten, alle Dateien zu überschreiben. Es müssen keine Änderungen an der Datenbank durchgeführt werden. Wenn Sie von Version 4.0-4.4 upgraden wollen, dann ist am einfachsten alle Dateien zu überschreiben und diese Datei zu löschen (außer Sie haben noch eigene Veränderungen daran vorgenommen):

  • includes/modules/YOUR_TEMPLATE/attributes.php

Wie benutze ich diese Erweiterung

Gehen Sie in Ihren Adminbereich --> Webshop --> Lagerbestand für Artikel mit Attributen, dort können Sie nun die entsprechenden Lagerbestände anlegen. Ebenso sollten Sie unter Konfiguration --> Lagerverwaltung Ihre Einstellungen überprüfen. Dort finden Sie auch eine neue Option die es Ihnen ermöglicht, dem Kunden den verfügbaren Lagerbestand anzeigen zu lassen, sollte er mehr Artikel als verfügbar bestellen.

Sprachliche Anpassungen dieser Erweiterung

Dieser Erweiterung liegen bereits die Sprachen Deutsch, Englisch und Französisch bei. Sollten Sie noch weitere Sprachen in Ihrem Shop verwenden, gehen Sie bitte wie folgt vor:

Kopieren Sie den Ordner german der Erweiterung aus dem Verzeichis admin/includes/languages und benennen Sie anschließend diese Kopie entsprechend Ihrer Sprache um (z.B. spanish). Danach öffnen Sie alle enthaltenen Dateien und verändern die entsprechenden Texte.
Das Wiederholen Sie bitte für den Ordner includes/languages/german/. Abschließend spielen Sie dann diese beiden Ordner in die entsprechenden Ordner auf Ihrem Webspace ein.

12. Low stock email now triggered by low stock on an attribute, even if overall stock for product remains above stock reorder level (new feature)
